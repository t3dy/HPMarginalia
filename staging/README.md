# Staging Directory

This directory contains intermediate artifacts from the dictionary enrichment and essay writing pipelines. These artifacts maintain a strict separation between:

1. **Retrieved evidence** (reading packets, essay data)
2. **Generated proposal text** (not currently stored here; generated inline during enrichment)
3. **Promoted content** (written to DB and site pages by enrichment scripts)

## Contents

### Reading Packets (`packets/`)
One JSON file per dictionary term, containing:
- Term metadata (slug, category, current review status)
- Retrieved corpus passages with full provenance (source document, chunk path, page refs, relevance score)
- Search terms used for retrieval

Generated by: `scripts/build_reading_packets.py`

### Essay Data
- `essay_russell.json`: Structured evidence for the Russell Alchemical Hands essay (DB queries + corpus search results)
- `essay_concordance.json`: Structured evidence for the Concordance Methodology essay (DB queries)

Generated by: `scripts/build_essay_data.py`

### Dictionary Audit
- `dictionary_audit.json`: Coverage report identifying terms with missing fields, weak terms, duplicate slugs, orphaned links

Generated by: `scripts/dictionary_audit.py`

## How to Run a Targeted Reading Pass

1. Run the dictionary audit to identify weak terms:
   ```
   python scripts/dictionary_audit.py
   ```

2. Build reading packets for specific terms or all DRAFT terms:
   ```
   python scripts/build_reading_packets.py                    # all DRAFT terms
   python scripts/build_reading_packets.py signature quire    # specific slugs
   ```

3. Enrich dictionary entries from packets:
   ```
   python scripts/enrich_dictionary.py
   ```

4. Rebuild the site:
   ```
   python scripts/build_site.py
   ```

## Data Status Model

All enrichment follows this provenance model:
- `source_method`: DETERMINISTIC | CORPUS_EXTRACTION | LLM_ASSISTED | HUMAN_VERIFIED
- `review_status`: DRAFT | REVIEWED | VERIFIED | PROVISIONAL
- `confidence`: HIGH | MEDIUM | LOW | PROVISIONAL

Retrieved evidence may populate staging artifacts automatically. LLM-generated interpretations must go into DRAFT fields only. No generated prose overwrites VERIFIED content.
